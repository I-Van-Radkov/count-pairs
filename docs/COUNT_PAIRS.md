### 1. Название
count_pairs

### 2. Синтаксис

Запуск приложения осуществляется с помощью следующей команды:
```bash
go run ./cmd/app/main.go <путь_к_файлу> <слово1> <слово2> <макс_расстояние>
```

### 3. Параметры

| Параметр | Описание |
| --- | --- |
| путь_к_файлу | Строка, указывающая путь к текстовому файлу для анализа. |
| слово1 | Первое слово для поиска вхождений в текстовом файле. |
| слово2 | Второе слово для поиска вхождений в текстовом файле. |
| макс_расстояние | Максимальное расстояние между словами, заданное целым неотрицательным числом. |

### 4. Описание

Приложение `count_pairs` предназначено для поиска количества вхождений пары слов в текстовом файле на заданном расстоянии друг от друга. Расстояние между словами определяется как количество слов между ними, при этом знаки препинания и переносы строк не учитываются. Поиск является регистронезависимым, пунктуация и переносы строк игнорируются. Уникальные пары вхождений учитываются один раз. Если слова совпадают, результат всегда равен нулю, так как одинаковые слова не могут образовывать пары.

### 5. Примеры

1. `go run ./cmd/app/main.go tests/data/Налоговый_кодекс.txt налоги России 5 -> 1`
   - В данном случае расстояние между словами «налоги» и «России» не превышает 5 слов, и существует одна уникальная пара вхождений.

2. `go run ./cmd/app/main.go tests/data/Налоговый_кодекс.txt налоги России 4 -> 0`
   - Расстояние между словами «налоги» и «России» превышает 4 слов, поэтому уникальных пар вхождений нет.

3. `go run ./cmd/app/main.go tests/data/Налоговый_кодекс.txt налоги доходы 1 -> 2`
   - Существует две уникальные пары вхождений слов «налоги» и «доходы», расстояние между которыми не превышает 1 слово.

4. `go run ./cmd/app/main.go tests/data/Налоговый_кодекс.txt налоги доходы 100 -> 2`
   - Несмотря на большое максимальное расстояние, существует только две уникальные пары вхождений слов «налоги» и «доходы».

5. Пример с разным регистром:
   ```bash
   go run ./cmd/app/main.go tests/data/Налоговый_кодекс.txt НАЛОГИ доходы 1 -> 2
   ```
   - Поиск является регистронезависимым, поэтому результат совпадает с примером 3.

6. Пример с игнорированием пунктуации:
   ```bash
   go run ./cmd/app/main.go tests/data/Налоговый_кодекс.txt налоги, доходы 1 -> 2
   ```
   - Пунктуация игнорируется, поэтому результат совпадает с примером 3.

### 6. Обработка ошибок

| Ситуация | Поведение |
| --- | --- |
| Неверное количество аргументов | Ошибка: требуется 4 аргумента |
| Отрицательное расстояние | Ошибка: максимальное расстояние должно быть неотрицательным целым числом |
| Нечисловое расстояние | Ошибка: максимальное расстояние должно быть неотрицательным целым числом |
| Файл не найден | Ошибка чтения файла |
| Нет прав доступа | Приложение выводит сообщение об ошибке и завершает работу |

### 7. Примечания

- Приложение поддерживает кодировку UTF-8.
- Временная сложность алгоритма составляет O(n).
- Ограничения: приложение работает только с текстовыми файлами, имеет ограничения по памяти и размеру файла.

### 8. Запуск тестов
